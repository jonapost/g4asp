Installing Geant4

1. Using a Geant4 virtual machine

This is the simplest option, as it avoids the effort to find and install the key pieces needed to create an installation on your own system.
  • Make sure that your computer meets the minimum requirements:
    • Windows or Linux users : desktop or laptop PC running Windows or Linux with a virtualization software installed
    • Mac users : desktop or laptop Mac running Mac OS with a virtualization software installed
    • 30 GB of free disk space available
    • at least 2 GB of RAM 'free' on the PC or Mac
    If the laptop does not meeting these requirements, the alternatives are a smaller virtual machine (option 2 below) or a native installation (option 3.)
  • Obtain a Virtual Machine 'player'. You can use either the free Virtual Box software (recommended) or VMware software: Workstation Player on Windows, or Fusion on Mac. The VMware software have evaluation versions (free for 30 days.) For further details see http://geant4.in2p3.fr/spip.php?article65 .
  • Download the Geant4 Virtual Machine from http://geant4.in2p3.fr/spip.php?rubrique8 or directly at ftp://ftp.cenbg.in2p3.fr/info/Vmware/geant4.10.03.p02/
  • PROBLEM: It is not clear exactly how to open the Image using Virtual box or VMware Fusion. Current instructions on opening images include:
    • how to open an OSboxes image using Virtual Box can be found at https://www.osboxes.org/guide/
    • It is not clear whether the Geant4 image includes any of these types of files.
    • Note: Don't forget to select 2048 MB of RAM memory.

2. Using a recent Linux Virtual Machine, and building Geant4 inside it

This is a hybrid method, which requires less disk space than the pre-built Geant4 virtual machine, ( approximately 5 GBytes ), and uses a recent Linux distribution. This is the preferred method for a Windows machine which has less available disk space (i.e. less than 40 GBytes. ) If you already have a recent Linux installation on your system, this may not be necessary - see whether option 3 works instead.
  • Obtain a Virtual Machine 'player'. See above (option 1) how to do this.
  • Obtain a recent version of Linux to run inside a Virtual Machine host:
    • I found such images at http://www.osboxes.org/
    • Some reasonable choices: Ubuntu 17.04 or 17.10, or Centos 7 .
    • To start a virtual machine, use the instructions at guide
    • To login use the credentials from the osboxes FAQ
  • Open a 'terminal' in your Linux installation
  • Check that you can find the compiler (gcc)
  • Proceed with the installation as in Option 3 - you have a working Linux distribution.

3. A Native Geant4 Installation


An Outline of the Installation Process
How to install Geant4 on your laptop or your desktop in your university/institute
If you have a Linux installation from the last 3-5 years, it should be possible to install a recent Geant4 (from the last 3 years) directly, without a virtual machine. To make sure that this is possible, see below for the requirements for the compiler version. The method is explained fully the Installation Instructions on the Geant4 web site. This document prepares you and gives a few shortcuts
  • First check the compiler that you have on your system
    • gcc --version on Linux
    • cl -von Windows
    • clang --version on MacOS
  • Choose the version of Geant4 to install on your system. You should choose a version of Geant4 which supports the compiler provided on your system. ( Some newer versions of Geant4 do not support compilers available on older systems. )
    • If the compiler and its version are supported by the latest Geant4 production release, then use it! As Geant4 10.4 is brand new ( December 2017 ), let's use Geant4 10.3 instead.
    • Geant4 10.3 requires gcc version 4.8.5 or higher, tested with 4.9.3, 5.3, 6.3
    • If you have gcc version 7.2, then use the newest version, Geant4 10.4 (as this is the only version that supports it.).
    • If you have a different subversion, e.g. 5.1, 5.2, 6.1, 6.2 or 7.1 it is likely but not certain that you it will work without a problem.
  • You can find the latest version of Geant4 at the at Download area or following the Download link in the home page of the Geant4 web site. The Release Notes document which compiler versions are supported. )
  • If your compiler is not supported, then check the previous versions of Geant4. A likely candidate is release 10.1 from December 2014, which did not require C++ 11 support, and thus supported older compilers.
    • For Geant4 10.3 or earlier versions, check the panel on the right of the Download page for the entry Previous Releases of Geant4 (since release 9.6)
    • The second most recent release is the top entry, so you many need to scroll down to find the older entries.
  • Important: you should always get the newest patch for the release you intend to use - for example there were 2 patches to release 10.2 in 2016.  Get the last one, now 10.2-patch-02

4. The installation process

  • The installation guide is the recommended way to install Geant4.
    • Locate the official installation guide in the "Support" pages of the Geant4 web site
      • for the latest release, which is currently Geant4 10.3 (patch 3), it is at http://geant4.web.cern.ch/geant4/UserDocumentation/UsersGuides/InstallationGuide/html/index.html
      • for previous versions of Geant4 you can find it at http://geant4.cern.ch/support/userdocuments.shtml
    • I suggest that you download the PDF version of the istallation guide for the version of Geant4 that you will use. After this you can search it - or just use it as backup.
    • I will not repeat information which is in the guide - this page is just an outline!
    • When you have a question, please look for an answer in the relevant section and then ask if you do not find an answer!
